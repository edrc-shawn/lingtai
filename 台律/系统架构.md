# 灵台系统架构 — LLM 变更导览

> 本文件定义灵台系统所有文件的角色、关系与变更规则。
> **改任何文件前，先读本文件确认关联影响。**

---

## 一、系统全景

```
用户指令（提炼/问知/体检/内观/产出）
       │
       ▼
AGENTS.md (零台入口——新AI用)
       │
       ▼
  ┌──────────────────────────────┐
  │  入口分发                     │
  │  WorkBuddy → skill/          │
  │  MiMo Code → /goal           │
  │  其他 AI   → AGENTS.md       │
  └──────────────┬───────────────┘
                 │
    ┌────────────┼────────────┐
    ▼            ▼            ▼
 提炼          问知         体检/内观
 (ingest)     (query)      (lint/内观)
```

## 三、变更影响面速查

### 改「台律/」文件时

```
运行手册.md  → 影响所有 skill 的执行规则
结构.md      → 影响开工skill、索引格式、创建页面时的字段
标签体系.md  → 影响体检脚本（lint_check.py）
MiMoGoal.md -> 需要手动复制给 MiMo Code
```

### 改「丹房/技能」文件时

```
灵台开工.md ↔ WB skill     → 同时更新两个位置
灵台问知.md ↔ WB skill     → 同时更新两个位置
```

### 新建丹房页时

```
1. 原料 → 已包含/已提炼 + 回链
2. 索引.md 追加一行
3. 日志.md 追加一行
4. Git commit
（交叉链接由每日检18:00执行）
```

### 改动任何文件名/路径时

```
1. 文件本身改名
2. 索引.md 更新引用
3. 结构.md（如果目录描述有变）
4. 搜索全库 [[旧路径]] → 更新为 [[新路径]]
5. 日志.md 记录
```

## 四、变更前自检清单（LLM 用）

**每次收到改文件指令，先快速过一遍：**

```
□ 这是改台律/ → 关联哪个 skill/MiMo goal？
□ 这是改技能（丹房/技能/核心）→ 同步到 WB skill 了吗？
□ 这是新建/改名页面 → 更新索引.md + 日志.md 了吗？
□ 这是删旧文件 → 索引.md 删了引用吗？结构.md 删了目录行吗？
□ 这次涉及 frontmatter → 规范是 2 字段白名单（处理状态/处理日期）
□ 这次涉及回链 → 放在 frontmatter 后、正文前，独占一行
□ 最后检查：日志.md 写了吗？git commit 了吗？
```

## 五、原料 frontmatter 规范（随时查阅）

```
---
处理状态: 待提炼 / 已包含 / 已提炼 / 已跳过 / 已过期 / 出错了
处理日期: YYYY-MM-DD
---

- → [[丹房/域/目标页]]    ← 不在 YAML 内，在 frontmatter 后、正文前
```

仅 2 字段白名单（处理状态/处理日期）。回链不在 YAML 内。
