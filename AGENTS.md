# AGENTS.md — 零台入口

> 任何 AI 编程助手读到本文件后，应按照灵台协议操作本知识库。

## 这是什么

灵台（Lingtai），基于 Obsidian 的 AI 知识管理系统。核心理念：**AI 整理弹药，人扣扳机。**

## AI 启动协议

新 LLM 会话接入后，按顺序读取：

1. `索引.md` — 全库导航
2. `台律/运行手册.md` — 操作规范
3. `台律/结构.md` — 格式规范
4. `入门/灵台用户画像.md` — 若存在，融入用户角色；不存在则以纯引擎模式运行

## 触发词 → 入口

| 触发 | 入口 |
|:----|:----|
| 提炼/ingest | `台律/运行手册.md` + `台律/结构.md` |
| 问知/query | `索引.md` + 目标丹房页 |
| 体检/lint | `python scripts/lint_check.py` |
| 产出 | 见 `入门/输出技能/` 下各 skill 文件 |

## 架构

```
灵台/
├── 台律/           ← AI 操作规范（运行手册/结构/标签体系/系统架构）
├── 入门/           ← 用户入门指南 + 画像模板
├── 丹房/           ← 提炼后的知识 Wiki
├── 原料/           ← 原始素材，只读
├── 体检/           ← Lint 检查 + 知识缺口
├── 索引.md         ← 全量目录
├── 日志.md         ← 操作日志
└── scripts/        ← 自动化脚本
```

## 核心约束

- 不编造：内容可追溯到原料原文
- 原料只读：仅标注处理状态，不改内容
- 融入优先：新知识融入已有页面，全新概念方建新页
- 操作收尾：写日志 → git commit

## 依赖

- Obsidian（推荐）
- Python 3.10+（体检脚本）
- 任意 AI 助手（WB/Claude/GPT/Copilot 均可）
